Introduction

Helium, a floating browser for Mac OS X now supports multiple windows.

This help will cover the few less obvious topics.

Key vs Current Window

With advent of multiple windows, it might not be obvious which is the current or key window, especially when Helium is not the default application.

The Helium key window features its title in a bold font.

When Helium is the current app, you can click on one of its windows to select a new key window.  To make Helium the current, active, application, hold down the SHIFT key when moving into a window.  You can also select a key window by using the Windows menu when Helium is active.

Note the mouse click on lower right window to make key window, but then moves outside to another, but key window is closed on user input ⌘W or ESCAPE key press.

Windows

While Helium supports multiple windows, these are not documents, not in the typical sense of individual storage, file / URL openings, or save dialogs.  Helium treats all documents as part of a larger context stored within the user's defaults.

Each of a document's details are either read-only attributes or read-write preferences.   Document details are automatically saved whenever a URL is made known to Helium or any changes are made, such a moving a document window.  Helium does support normal URL file open and drag-and-drop operations but not save.

Document Details

  • URL
  • attributes: window size, origin, time - for audio / video URLs
  • preferences: auto-hide, float-above, opacity, translucency
  • WebView - Mac OS X built-in web browser

URL

A window is presented for each URL made known to Helium by either filesystem or web location.   All URL access is read only.  Helium can write by drag-and-drop to the Finder. Dragging a document window title bar icon creates an alias to the document url.

Attributes

Name

A window name is derived from the document URL.

Window Size

By default a window's size is remembered when recalled.

Window Origin

By default a window's origin is remembered when recalled and updated when moved.

Duration

For video and audio URLs, Helium will catalog the asset's duration.

WebView

Each document window is a browser window and by its contextual menu, can be alter its attributes such as mute, loop, hide controls, copy video adress, and when implemented: full screen, and download.  It can also provide access to helper applications such as Share, as well as access to open new URLs, close window, and access to document attributes, preferences as well as to access playlists and their play items.

Preferences

Auto-hide

Helium can hide and restore a document window's title when moving the pointer beyond the document window.  This preference is document specific.

Float Above

Helium will float its document windows above other apps, across all spaces. Optionally the window can be local to the current space.&ebsp; Note that windows cannot be moved across spaces.  This preference is document specific.

Opacity

Helium can vary a document window opacity value from 10 .. 100%.   This preference is document specific.

History Keep

Helium will keep the items seen in a history playlist named "History" by default; this may be altered in the Playlists panel.

The default history "keep" size is 256 items. Should you need to alter the default use the Terminal app; for example:

    1% defaults read -app Helium historyKeep
    256
    1% defaults write -app Helium historyKeep 100
    1% defaults read -app Helium historyKeep
    100

It is not advised to choose a value too large.

Zoom Icon

Helium window support full screen - aka "zoom" but does not display a zoom button by default it its title bar. The preferecce be altered by using the Terminal app; for example:

    1% defaults read -app Helium hideZoomIcon
    true
    1% defaults write -app Helium hideZoomIcon false
    1% defaults read -app Helium hideZoomIcon
    true

This value is only read at startup.

Playlist Throttle

Helium will limit the size of a playlist "play" to reduce runaway sessions.

Should you need to alter the default use the Terminal app; for example:

    1% defaults read -app Helium playlistThrottle
    32
    1% defaults write -app Helium playlistThrottle 16
    1% defaults read -app Helium playlistThrottle
    16

It is not advised to choose a value too large.

Translucency

Helium can vary a document window opacity display when moving the pointer inside or outside a document window, never, or always.  This preference is document specific.  This video shows from top left clockwise, an auto-hide title, an always transluency, a mouse over, a mouse outside, and a never translucency window.

History

Helium remembers all documents seen in a history queue.   This information is saved and restored across sessions.

Search

Helium, provides a web search interface to Bing, Google and Yahoo! web search urls.

Searches are cached within the default history playlist and preserved across sessions.

Playlists

Helium can catalog documents into playlists, groups of document URLs, which can be restored or opened by play list or its individual play items.  By default, Helium will restore all document windows left open from a prior session. Playlist items can be manually created or via drag-n-drop. Editing a playlist item is guarded by its non-zero time field.

The tableView corner menu features a button to renumber in current order playlist items. This ranking governs which item are selected for playlist. Duplicate items can be pruned by holding down SHIFT key while pressing the button.

Menus

In addition to the open, and load file or location menu items, Helium supports a recents menu

a dock menu

a status bar menu

and a contextual menu.
